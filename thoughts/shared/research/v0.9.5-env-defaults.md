## ¬ßA ‚ÄĒ Collisions to resolve in Phase 1

Both sides declare `ENABLE_ONEDRIVE_PERSONAL` and `ENABLE_ONEDRIVE_BUSINESS`. The resolution is **keep our `PersistentConfig` wrapper** (admin-editable from the Cloud Sync tab ‚ÄĒ see Confluence Cloud Sync work in `state.md`), optionally with upstream's `and bool(ONEDRIVE_CLIENT_ID_*)` guard added inside the wrapper.

### Our version (`backend/open_webui/config.py:2914`)

```python
ENABLE_ONEDRIVE_PERSONAL = PersistentConfig(
    'ENABLE_ONEDRIVE_PERSONAL',
    'onedrive.enable_personal',
    os.environ.get('ENABLE_ONEDRIVE_PERSONAL', 'True').lower() == 'true',
)
ENABLE_ONEDRIVE_BUSINESS = PersistentConfig(
    'ENABLE_ONEDRIVE_BUSINESS',
    'onedrive.enable_business',
    os.environ.get('ENABLE_ONEDRIVE_BUSINESS', 'True').lower() == 'true',
)
```

### Upstream version (`upstream/main:config.py:2679`)

```python
ENABLE_ONEDRIVE_PERSONAL = os.environ.get('ENABLE_ONEDRIVE_PERSONAL', 'True').lower() == 'true' and bool(
    ONEDRIVE_CLIENT_ID_PERSONAL
)
ENABLE_ONEDRIVE_BUSINESS = os.environ.get('ENABLE_ONEDRIVE_BUSINESS', 'True').lower() == 'true' and bool(
    ONEDRIVE_CLIENT_ID_BUSINESS
)
```

### Recommended resolution (Lex decision 2026-05-25: keep ours, no upstream guard)

Keep `PersistentConfig` wrapper unchanged ‚ÄĒ Cloud Sync admin tab already prevents flipping these flags on without a configured client id, so upstream's `and bool(client_id)` guard would be redundant and adds friction. **Action for Phase 1:** keep `--ours` for these two PersistentConfig blocks; do **not** adopt the upstream guard.

## ¬ßC ‚ÄĒ Plan table corrections

The Phase 0 step-3 table in the plan referenced three vars that don't quite match upstream:

1. **`AUDIO_STT_SKIP_PREPROCESSING`** doesn't exist in upstream. The actual env var is **`BYPASS_PYDUB_PREPROCESSING`** in `env.py:578` (also affects TTS, not just STT). Rename when updating `.env.example`.
2. **`OPEN_WEBUI_SECURITY_HEADERS`** doesn't exist as a single var. Upstream's security-header mechanism reads ~15 individually-named env vars inside `utils/security_headers.py::set_security_headers()` ‚ÄĒ see ¬ßB2.
3. **`AIOHTTP_*` pool tuning vars** are 3 specific names ‚ÄĒ listed in ¬ßB1 with their defaults.

    'ENABLE_AUTOMATIONS',
